## Инструкция по приватной ветке (на русском)

Если вы хотите держать будущие премиальные кейсы приватными, сохраняя при этом публичный основной репозиторий, следуйте этой простой инструкции:

1. Создайте приватную ветку локально или используйте отдельный приватный репозиторий. Рекомендуемое название ветки: `private-cases`.

```bash
# создать и переключиться на приватную ветку
git checkout -b private-cases
```

2. Работайте с приватными кейсами в отдельной папке (рекомендуется: `private_cases/`) и НЕ пушьте эти файлы в публичную ветку `main`.

```bash
# создать директорию для приватных кейсов и сгенерировать шаблон (dry-run)
mkdir -p private_cases
node -r ts-node/register scripts/new-case-template.ts -o private_cases --dry-run
# когда будете готовы — записать файл без dry-run
node -r ts-node/register scripts/new-case-template.ts -o private_cases --id PREMIUM-001
git add private_cases/PREMIUM-001.json
git commit -m "Add private premium case PREMIUM-001"
```

3. Запушьте приватную ветку в приватный удалённый репозиторий (рекомендуется) или храните её локально/в защищённом месте.

```bash
# push приватной ветки на удалённый репозиторий (требуются права доступа)
git push origin private-cases
```

4. Если вам необходимо хранить приватные кейсы в том же удалённом репозитории, используйте средства управления доступом (protected branches, team permissions) или отдельный приватный форк.

Замечания:
- Скрипты в `scripts/` поддерживают указание директории через `-d|--dir` или переменную окружения `CASES_DIR`, поэтому вы можете указывать `private_cases` при работе локально.
- Пример: `CASES_DIR=private_cases npm run validate` выполнит валидацию приватной директории с кейсами.
